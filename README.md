# SE_DAY4_Software-Project-Management


                                        ANSWERS

 1. Delivering on time keeps customers happy and avoids extra costs. Project managers can set clear goals, track progress often, and adjust resources when needed.

2. Good cost control avoids waste and keeps the project profitable. You can use a detailed budget, plan for surprises, and check expenses regularly.

3. Agile is flexible and adapts quickly but can lack structure. Waterfall is more organized but less flexible. Agile fits changing projects; Waterfall fits fixed, clear ones.


4. Agile works well for mobile or web apps that change often. Waterfall is better for critical systems like medical or banking software.


5. Regular testing, code reviews, and automation help ensure quality. High standards reduce bugs and make customers happier.

6. Defining the project scope makes planning easier. A Work Breakdown Structure (WBS) breaks work into small tasks, making it easier to manage.

7. A detailed schedule helps avoid delays and track progress. Gantt charts show tasks, deadlines, and how they depend on each other.

8. My software helps agencies manage and rent cars easily. This solves their need for fast and simple booking systems.

9. When the problem is clear, the solution can be more accurate and useful, with no wasted features.

10. It’s a simple platform that helps agencies manage and rent their vehicles online.

11. Real-time fleet tracking, online booking, and a dashboard with statistics make it stand out.

12. The digital rental market is growing fast, and more agencies want to go digital.

13. Knowing market trends helps you build software that fits current needs and attracts more users.
